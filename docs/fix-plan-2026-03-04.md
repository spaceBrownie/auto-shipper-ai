## Fix 1: Local Dev Stub Providers

**Problem:** `POST /verify-costs` requires live FedEx/UPS/USPS/Stripe/Shopify API keys. No fallback for local dev.

**Approach:** Add `@Profile("local")` stub implementations for all 5 external providers. Mark live adapters with `@Profile("!local")` so they don't load in local profile.

### Files to create:
- `modules/catalog/src/main/kotlin/com/autoshipper/catalog/proxy/carrier/StubCarrierRateProvider.kt`
  - Implements `CarrierRateProvider`, returns hardcoded rates (~$5-8 USD)
  - `@Component @Profile("local")` — registers 3 stub beans (UPS, FedEx, USPS) via a `@Configuration` class
- `modules/catalog/src/main/kotlin/com/autoshipper/catalog/proxy/payment/StubStripeProcessingFeeProvider.kt`
  - Mirrors `StripeProcessingFeeProvider` — applies 2.9% + $0.30 formula without API call
  - `@Component @Profile("local")`
- `modules/catalog/src/main/kotlin/com/autoshipper/catalog/proxy/platform/StubShopifyPlatformFeeProvider.kt`
  - Returns 2.0% of estimated order value (Basic plan)
  - `@Component @Profile("local")`

### Files to modify:
- `modules/catalog/src/main/kotlin/com/autoshipper/catalog/proxy/carrier/FedExRateAdapter.kt` — add `@Profile("!local")`
- `modules/catalog/src/main/kotlin/com/autoshipper/catalog/proxy/carrier/UpsRateAdapter.kt` — add `@Profile("!local")`
- `modules/catalog/src/main/kotlin/com/autoshipper/catalog/proxy/carrier/UspsRateAdapter.kt` — add `@Profile("!local")`
- `modules/catalog/src/main/kotlin/com/autoshipper/catalog/proxy/payment/StripeProcessingFeeProvider.kt` — add `@Profile("!local")`
- `modules/catalog/src/main/kotlin/com/autoshipper/catalog/proxy/platform/ShopifyPlatformFeeProvider.kt` — add `@Profile("!local")`
- `modules/catalog/src/main/kotlin/com/autoshipper/catalog/config/ExternalApiConfig.kt` — add `@Profile("!local")` (RestClient beans not needed locally)

**Note:** `StripeProcessingFeeProvider` and `ShopifyPlatformFeeProvider` are not behind interfaces — `CostGateService` depends on them directly. The stubs need to extend the concrete classes OR we extract interfaces. Simpler: the stubs extend the concrete classes and override the methods. But the concrete classes inject `RestClient` and `@Value` props in the constructor. Cleanest approach: **extract simple interfaces** (`ProcessingFeeProvider`, `PlatformFeeProvider`), have both live and stub implement them, and update `CostGateService` to depend on the interfaces.

### Interface extraction:
- Create `modules/catalog/src/main/kotlin/com/autoshipper/catalog/proxy/payment/ProcessingFeeProvider.kt` — interface with `fun getFee(estimatedOrderValue: Money): Money`
- Create `modules/catalog/src/main/kotlin/com/autoshipper/catalog/proxy/platform/PlatformFeeProvider.kt` — interface with `fun getFee(): Money`
- Update `StripeProcessingFeeProvider` to implement `ProcessingFeeProvider`
- Update `ShopifyPlatformFeeProvider` to implement `PlatformFeeProvider`
- Update `CostGateService` constructor: `StripeProcessingFeeProvider` → `ProcessingFeeProvider`, `ShopifyPlatformFeeProvider` → `PlatformFeeProvider`

## Fix 2: Wire Initial Pricing on SKU Listing

**Problem:** `GET /api/skus/{id}/pricing` returns 404 because `PricingEngine.setInitialPrice()` is never called.

**Approach:** Add an `@EventListener` in the pricing module that reacts to `SkuStateChanged` when `toState == "LISTED"` and calls `setInitialPrice()`.

### File to create:
- `modules/pricing/src/main/kotlin/com/autoshipper/pricing/domain/service/PricingInitializer.kt`
  - `@Component` with `@EventListener` for `SkuStateChanged`
  - When `toState == "LISTED"`: look up the stress test result (estimated price, stressed total cost) and cost envelope, compute margin, call `PricingEngine.setInitialPrice()`
  - Depends on: `PricingEngine`, `CostEnvelopeRepository`, `StressTestResultRepository` (cross-module read via public JPA entities — consistent with existing pattern in `PricingEngine` which already imports `CostEnvelopeRepository`)
  - Guard: skip if price already exists (idempotent)

### Key data flow:
1. `SkuStateChanged(toState = "LISTED")` fires
2. `PricingInitializer` reads `StressTestResultEntity` for `estimatedPriceAmount` and `stressedTotalCostAmount`
3. Computes margin: `(price - cost) / price * 100`
4. Calls `PricingEngine.setInitialPrice(skuId, price, margin, fullyBurdenedCost)`

### Files to modify:
- `modules/pricing/build.gradle.kts` — add `implementation(project(":catalog"))` if not already present (it is — already depends on `:catalog`)

## Fix 3: Map Domain Exceptions to HTTP Errors

**Problem:** `InvalidSkuTransitionException` returns 500. `ProviderUnavailableException` returns 500. `NoSuchElementException` (SKU not found) returns 500.

**Approach:** Add a `@ControllerAdvice` exception handler in the `:catalog` module.

### File to create:
- `modules/catalog/src/main/kotlin/com/autoshipper/catalog/handler/GlobalExceptionHandler.kt`
  - `@ControllerAdvice` class
  - `InvalidSkuTransitionException` → 409 Conflict with `{"error": "...", "message": "..."}`
  - `ProviderUnavailableException` → 502 Bad Gateway with `{"error": "...", "message": "..."}`
  - `NoSuchElementException` → 404 Not Found with `{"error": "...", "message": "..."}`
  - `StressTestFailedException` — already handled in controller, skip here
