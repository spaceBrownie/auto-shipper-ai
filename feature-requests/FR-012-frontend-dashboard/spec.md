# FR-012: Frontend Dashboard

## Problem Statement

Operators need real-time visibility into the commerce engine's portfolio performance, SKU health, vendor status, and capital metrics. Without a dashboard, the system is a black box — margins can degrade, vendors can underperform, and capital can be misallocated without anyone noticing. The frontend must surface the data generated by the backend in a clear, actionable interface.

## Business Requirements

- Portfolio overview: total experiments, active SKUs, terminated SKUs, blended portfolio margin, capital deployed
- SKU management: list all SKUs by state, view individual SKU detail (cost envelope, stress test result, pricing history, P&L)
- Vendor dashboard: list vendors, view reliability scores, SLA breach history, linked SKUs
- Capital dashboard: rolling reserve balance, SKU-level P&L, margin trend charts, shutdown rule status
- Experiment tracker: active experiments, budget consumption, validation window countdown
- All data is read-only in Phase 1 — no write operations from the frontend except approvals
- Human approval flow for: scaling decisions, capital reallocation, vendor reinstatement after breach

## Success Criteria

- React + Vite + shadcn/ui project initialized and building
- Portfolio summary page renders live data from `GET /api/portfolio/summary`
- SKU list page with state filter renders from `GET /api/skus`
- SKU detail page with cost breakdown, stress test result, pricing history, P&L chart
- Vendor list with reliability scores from `GET /api/vendors`
- Capital dashboard with reserve balance and margin trend from `GET /api/capital/reserve`
- All pages handle loading, error, and empty states
- Deployed and accessible at `http://localhost:5173` in development

## Non-Functional Requirements

- TypeScript strict mode enabled
- React Query (TanStack Query) for all data fetching and caching
- Recharts or shadcn/ui charts for margin trend visualization
- Responsive layout — usable on 1280px+ screens
- No authentication in Phase 1 (internal tool, not public-facing)

## Dependencies

- FR-002 (project-bootstrap) — backend must be running and CORS configured
- FR-003 (catalog-sku-lifecycle) — SKU list and detail APIs
- FR-006 (pricing-engine) — pricing history API
- FR-007 (vendor-governance) — vendor APIs
- FR-009 (capital-protection) — P&L and reserve APIs
- FR-010 (portfolio-orchestration) — portfolio summary and experiment APIs
